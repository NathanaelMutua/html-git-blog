What is Git

Git is the most popular version control system, it really helps us keep track of different versions of a given project or document.

Introduction to Version Control

Version control, as it says itself is all about keeping track of versions - That's prolly how I'd put it in laymans terms.

Version control systems, which I found out are sometimes called source code management(SCM) or revision control systems(RCS)

Version control allows many members to work collaboratively on a project, even of they are not in the same place.

Versioning

As I've come to learn versioning is the creation and management of multiple product releases

This is signified through versioning numbers, such as: v2.2.1

Meaning of each digit
  • "v2..2.1": This is a MAJOR change in a version
  • "v2.2.1": This usually represents a patch release, usually having minor features.
  • "v2.2.1": This represents a patch release, it is usually just basic bug fixes

History

Git was created by Linus Torvald, the creator of the Linux Kernel, in 2005 in response to a liscence dispute with BitKeeper - a version control system at the time. Linus Torvald created Git in 10 days to provide a free, fast alternative to BitKeeper

On December 21st, 2005, Git had its first major release.

Today(on the day of publishing) the latest version of Git is 2.49.0

Why Use Git

  1. Helps us keep track of changes made by different members
  2. Tracking of changes makes going back to a stable version easier
  3. It ensures better communication and organization
  4. Allows for many members to work on the same project simultaneously
  5. Branching and merging allows people to create seperate branches, which they can work on seperately, before requesting to be merged to the main branch.

Installing Git

As indicated in the Git documentation, one can download as a package or via another installer. I highly recommend the autommated installation.

The instructions are as follows:

  1. Visit the official website: Downloading Git for Windows
  2. Select the standalone installer (If you want to download onto your desktop) or the portable edition (if you want to have the installer in a seperate or external storage)
  3. After the installer file has been downloaded, execute it
  4. Usually a system permission pop up appears - Allow it, then follow the following step-by-step instructions
    • 1: Information-Public liscence page: Click next for this section - I know I did
    • 2: Review selected components: This is based on preference, but ensure all the necessary boxes are checked
    • 3: Choosing the default editor used by Git: I preferably leave it as is, because we can configure it later using
    • 4: Adjusting name of initial branch: This is yet again about preference, but many teams are migrating to main as the default branch
    • 5: Adjusting PATH environment: I'd leave it as default, but take note of it, as we will probably need it for troubleshooting purposes
    • 6: Choosing SSH executable: I went with the Bundled Open SSH, but I've come to learn that if one needs specific features, has existing setup, needs troubleshooting, then External OpenSSH comes in clutch
    • 7: Choosing HTTPS transport backend: I'd leave as is
    • 8: Configuring line ending conversions: I'd leave as is
    • 9: Configuring the terminal emulator to use with Git Bash: I'd leave as is
    • 10: Choosing default behavior of git pull: I'd leave as is by default
    • 11: Choosing a credential helper: I'd leave as is
    • 12: Configuring extra options, these are cache options: Decide as you may prefer
    • 13: Installation: Lastly click on the 'Install' button
  5. I learnt that you can also just run this 'sudo dnf install git-all' command in the command prompt(As Administrator)
